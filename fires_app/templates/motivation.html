  • Each day, there are hundreds of satellite images from NASA's AQUA and TERRA satellites that are run through the MODIS active fire product. The MODIS active fire product detects fires in 1km pixels that are burning at the time of overpass under relatively cloud-free conditions using a contextual algorithm, where thresholds are first applied to the observed middleâ€“infrared and thermal infrared brightness temperature and then false detections are rejected by examining the brightness temperature relative to neighboring pixels (Giglio, L. et al. 2003). The output of this active fire product is a data set that holds hundreds of 'detected fires' at given latitude/longitude coordinates.

    Currently, this detected fires data set is not used in real-time, or even near real-time, for forest fire prevention. While this could be due to the processing time it takes to analyze the AQUA and TERRA images, the large number of false positives with regards to forest-fires doesn't help. The MODIS active fire product detects not only forest fires, but also house fires, farmer burn piles, hot asphalt parking lots, and other hot surfaces that could potentially be forest fires. Unfortunately, there is no way to tease out which of these detected fires are forest fires in real-time; current practice is to use the detected fires dataset to confirm daily fire perimeter boundaries submitted by state forest services agencies.

    Using this detected fires data set in combination with historical fire perimeter boundaries and weather data, I aim to build a model that can identify which of these detected fire are actually forest fires. My hope is that I can achieve a high enough level of accuracy that the detected fires data set can be used in near real-time to aid forest fire prevention, where every minute counts.
